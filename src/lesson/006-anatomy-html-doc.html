<!doctype html>

A required preamble used to define the HTML version. The current HTML version is 5.

<html lang="en">

The opening tab of the root element that wraps all the content of document.

<head>

Container element for content you want to include on the HTML page that isn't the content you are showing to your page's viewers. This includes things like keywords and a page description that you want to appear in search results, CSS to style content, character set declarations, and more.

  <title>Home</title>

The title of the page. This is what appears in the browser tab.

  <meta charset="UTF-8" />

The character set of the page. UTF-8 is the default character set that supports most languages.

  <meta name="viewport" content="width=device-width" />

Viewport means the part of the web page that you can see. The viewport meta tag tells the browser how to scale the page to fit the device. The width="device-width" part sets the width of the page to follow the screen-width of the device (which will vary depending on the device).

  <link rel="stylesheet" href="styles.css" />

This is how you link to a CSS file. The rel="stylesheet" part tells the browser that this is a stylesheet. A stylesheet is a file which tells a browser how to render a page The href="styles.css" part tells the browser where to find the CSS file. href stands for hypertext reference.

  <script type="module" src="scripts.js"></script>

This is how you link to a JavaScript file. The type="module" part tells the browser that this is a JavaScript module, and can use modern syntax like the import keyword. A JavaScript module is a file which contains JavaScript code. The src="scripts.js" part tells the browser where to find the JavaScript file.

</head>

The closing tag of the head element. This is how you tell the browser that you are done adding content to the head element.

<body>

Container element for content you want to include on the HTML page that is the content you are showing to your page's viewers. This includes things like text, images, videos, and more.

  <main>

The dominant content of a <body> element. There should be only one in a document, but it doesn't have to be the direct descendent of the <body> element. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms. It should never be the descendent of a <article>, <aside>, <header>, <footer>, <nav>, or <section> element.

    Hello World!

The content of the page. This is what appears in the browser.

  </main>

The closing tag of the <main> element.

</body>

The closing tag of the <body> element.

</html>

The closing tag of the root element. The end of the document.
